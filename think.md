## React感想

1. 很遗憾，其实除了公用组件，大部分业务组件都没有办法做到令人满意的复用程度。

2. 要注意react合成事件和不可避免调用原生绑定事件时候的冲突。

3. Ajax拿到的数据到底放在组件內还是状态管理库？如果是组件内，应该放在props还是staete 这势必导致组件无法复用，而放在状态管理库的话会不会导致层级过深，同样需要组件发起action。

4. 理想下的store库管理下的react应用，组件应分为容器组件和视图组件，容器组件应当负责从store拿到数据，视图组件的数据从props获取。

5. react真是美啊，同样的数据映射成唯一视图，这种函数式的对应法则像极了声明式的代码写法，完全背离了传统的过程式编码。

```
function renderView(data){
    return pureFunction(data);
}
```
